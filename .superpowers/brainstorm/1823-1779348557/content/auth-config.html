Section 3 of 4 — Credentials, tokens, and setup

SuccessFactors Auth — OAuth2 SAML Bearer

SF uses a 2-step handshake. The agent handles this automatically — users never see it.

Step 1 Build a SAML assertion, sign it with your X.509 private key
Step 2 POST assertion → SF token endpoint → receive OAuth2 access token
Step 3 Attach token to every API call: Authorization: Bearer <token>
Auto Token cached in memory, refreshed before expiry (default TTL: 12h)

.env File — Everything in One Place

# ── SuccessFactors ──────────────────────────────
SF_BASE_URL=https://api4.successfactors.com   # your SF API host
SF_COMPANY_ID=your_company_id
SF_USER_ID=admin_user_id
SF_CLIENT_ID=your_api_key_from_sf
SF_PRIVATE_KEY_PATH=./certs/private_key.pem   # X.509 private key

# ── LLM Provider ────────────────────────────────
LLM_PROVIDER=openai          # openai | anthropic | azure | ollama | gemini | ...
LLM_MODEL=gpt-4o             # model name for the chosen provider
LLM_API_KEY=sk-...           # API key (leave blank for local Ollama)
LLM_API_BASE=                # optional: custom endpoint (Azure, proxy, etc.)

# ── Bot Integrations (optional) ─────────────────
TELEGRAM_BOT_TOKEN=          # from @BotFather
WHATSAPP_TOKEN=              # Meta WhatsApp Business API token
WHATSAPP_PHONE_ID=           # Meta phone number ID
