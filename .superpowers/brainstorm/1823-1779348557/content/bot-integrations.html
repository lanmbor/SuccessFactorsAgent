Bot Integrations

Section 4 of 4 — Telegram and WhatsApp access to the same agent

Shared Agent Core

All three frontends funnel messages into the same agent core. Bots are thin adapters — they translate platform messages into the agent's input format and route replies back.

Telegram

Library
python-telegram-bot v21+
Async, supports both polling (local) and webhooks (cloud). Polling mode works out-of-the-box with no public URL needed.
Modes
Local: polling (no server needed)
Cloud: webhook → FastAPI endpoint
Switched automatically based on TELEGRAM_WEBHOOK_URL in .env

WhatsApp

Via Meta Cloud API
WhatsApp Business Platform (free tier available)
Requires: public HTTPS URL for webhook verification. Best for cloud deployment.
Via Twilio (alternative)
Twilio WhatsApp sandbox
Easier to set up locally with ngrok. Paid service but simpler API. Good for testing.

Message Flow

Telegram/WhatsApp → Adapter extracts text + user ID
Adapter → Calls agent.chat(user_id, message) (same as Chainlit)
Agent → Runs ReAct loop, calls SF tools as needed
Agent → Returns plain text response
Adapter → Sends reply back via Telegram/WhatsApp API

Per-User Conversation Memory

Each user (identified by platform + user ID) gets their own conversation history stored in memory during the session. Chainlit handles this automatically; the bots replicate the same pattern with an in-memory dict keyed by platform:user_id.
